# Техническая документация (Пример 1)

## Введение

Этот раздел служит введением в функцию `process_payment`, которая является основным компонентом для расчёта итоговой суммы платежа с учётом возможных скидок. Документация предназначена для junior-разработчиков, чтобы помочь им быстро понять назначение, использование и внутреннюю логику этой функции.

Функция `process_payment` предназначена для вычисления конечной суммы к оплате после применения скидки. Она принимает исходную сумму платежа (`amount`) и, опционально, размер скидки (`discount`). Скидка выражается в виде десятичной дроби (например, 0.1 для 10%, 0.25 для 25%).

Ключевые особенности этой функции включают:
*   **Расчёт скидки:** Автоматическое применение указанной скидки к исходной сумме.
*   **Валидация ввода:** Проверка того, что исходная сумма платежа является положительным числом, предотвращая некорректные расчёты.

Эта функция может быть использована в различных сценариях, таких как системы электронных платежей, корзины покупок в интернет-магазинах или любые другие приложения, где требуется гибкий механизм обработки платежей с возможностью применения скидок. Далее в документации мы подробно рассмотрим параметры функции, её возвращаемое значение, обработку ошибок и примеры использования.

## Параметры

Функция `process_payment` разработана для гибкого расчёта итоговой суммы платежа, и для этого она принимает два ключевых параметра: `amount` и `discount`. Понимание их назначения и ожидаемых значений критически важно для корректного использования функции.

### `amount`

*   **Тип**: `float` или `int` (числовое значение).
*   **Описание**: Представляет собой исходную (первоначальную) сумму платежа. Именно от этой суммы будет рассчитываться и вычитаться скидка.
*   **Ожидаемые значения и ограничения**:
    *   Параметр `amount` должен быть **положительным числом** (строго больше нуля).
    *   Если вы передадите значение, которое меньше или равно нулю (`0` или отрицательное число), функция вызовет ошибку `ValueError`. Это сделано для предотвращения логически некорректных расчётов платежей.

### `discount`

*   **Тип**: `float` или `int` (числовое значение).
*   **Описание**: Определяет процент скидки, который будет применён к `amount`. **Важно**: скидка выражается в виде десятичной дроби.
    *   **Примеры**:
        *   Чтобы применить скидку в 10%, передайте `0.1`.
        *   Для 25% скидки используйте `0.25`.
        *   Если скидки нет, можно передать `0.0` или не указывать этот параметр вовсе.
*   **Значение по умолчанию**: `0`. Это означает, что если при вызове функции вы не укажете значение для `discount`, функция автоматически применит скидку в 0%. Другими словами, сумма останется неизменной.
*   **Ожидаемые значения и рекомендации**:
    *   Обычно ожидается, что значение `discount` будет находиться в диапазоне от `0` (нет скидки) до `1` (100% скидка).
    *   Хотя код технически позволяет передавать значения вне этого диапазона, это может привести к неочевидным результатам: например, итоговая сумма может стать отрицательной (при `discount > 1`) или даже увеличиться (при `discount < 0`). Для предсказуемого поведения функции рекомендуется всегда передавать `discount` в диапазоне от `0` до `1`.

## Возвращаемое значение

Функция `process_payment` возвращает **итоговую сумму платежа** после применения всех скидок.

*   **Тип данных**: `float` (число с плавающей точкой). Даже если вы передадите целые числа (`int`) в качестве `amount` и `discount`, результат может быть числом с десятичной частью (например, `90.0`), особенно если скидка не является целым процентом. Поэтому рекомендуется всегда ожидать `float`.
*   **Описание**: Возвращаемое значение представляет собой окончательную сумму, которую необходимо фактически оплатить. Оно вычисляется путём вычитания применённой `discount` (скидки) из исходной `amount` (суммы).
*   **Пример**:
    *   Если `amount` равно `100` и `discount` равно `0.1` (что соответствует 10% скидки), функция вернёт `90.0`.
    *   Если `amount` равно `50` и `discount` равно `0` (нет скидки), функция вернёт `50.0`.

Важно отметить, что возвращаемое значение всегда будет неотрицательным числом, так как исходная `amount` должна быть больше нуля, а скидка уменьшает эту сумму.

## Возможные ошибки

При работе с функцией `process_payment` вы можете столкнуться с определёнными ошибками. Понимание их причин поможет вам правильно обрабатывать исключительные ситуации и писать более надёжный код.

### 1. `ValueError`

Функция `process_payment` явно проверяет одно важное условие: сумма платежа (`amount`) должна быть положительным числом.

*   **Причина:** Если вы передадите значение для параметра `amount`, которое равно `0` или меньше `0` (например, отрицательное число).
*   **Когда возникает:** Внутри функции сработает условие `if amount <= 0:`, и будет сгенерировано исключение `ValueError` с сообщением "Сумма должна быть больше нуля".
*   **Почему это важно:** Платеж не может быть нулевым или отрицательным в реальных сценариях, поэтому функция предотвращает такие некорректные расчёты.
*   **Пример:**
    ```python
    # Это вызовет ValueError
    # process_payment(0)
    # process_payment(-50)
    ```
*   **Как избежать/обработать:** Всегда убеждайтесь, что `amount` является положительным числом перед вызовом функции, или используйте блок `try-except` для перехвата `ValueError`:
    ```python
    try:
        final_amount = process_payment(-100, 0.1)
        print(f"Итоговая сумма: {final_amount}")
    except ValueError as e:
        print(f"Ошибка: {e}") # Выведет "Ошибка: Сумма должна быть больше нуля"
    ```

### 2. `TypeError`

Хотя функция не проверяет типы данных явно, Python сгенерирует `TypeError`, если вы передадите параметры, которые не являются числами.

*   **Причина:** Если `amount` или `discount` будут строками, списками или любыми другими нечисловыми типами данных, с которыми нельзя выполнять арифметические операции (сложение, вычитание, умножение).
*   **Когда возникает:** Python попытается выполнить операцию, например, `amount - (amount * discount)`, и обнаружит, что операнды несовместимы.
*   **Пример:**
    ```python
    # Это вызовет TypeError
    # process_payment("100", 0.1)
    # process_payment(500, "10%")
    ```
*   **Как избежать:** Всегда передавайте числовые значения (`int` или `float`) для параметров `amount` и `discount`. Если вы получаете данные из пользовательского ввода или из внешних источников (например, из формы или API), всегда преобразуйте их к числовому типу перед передачей в функцию.

## Примеры использования

Этот раздел демонстрирует, как применять функцию `process_payment` в различных сценариях. Вы увидите, как рассчитывать итоговую сумму платежа как со скидками, так и без них, а также как функция обрабатывает различные типы входных данных.

### 1. Расчёт платежа без скидки

Если скидка не применяется, вы можете просто передать сумму платежа. В этом случае будет использовано значение по умолчанию для параметра `discount`, которое равно `0`.

```python
# Пример 1: Платеж на 100 единиц без скидки
amount_1 = 100
final_amount_1 = process_payment(amount_1)
print(f"Итоговая сумма для {amount_1} без скидки: {final_amount_1}")
# Ожидаемый вывод: Итоговая сумма для 100 без скидки: 100.0
```

*Пояснение*: В данном примере `process_payment(100)` эквивалентно вызову `process_payment(100, 0)`, так как `discount` по умолчанию равен `0`.

### 2. Расчёт платежа с процентной скидкой

Чтобы применить скидку, передайте её значение в виде десятичной дроби (например, `0.1` для 10%, `0.25` для 25%) вторым параметром `discount`.

```python
# Пример 2: Платеж на 200 единиц со скидкой 15% (0.15)
amount_2 = 200
discount_2 = 0.15  # 15% скидки
final_amount_2 = process_payment(amount_2, discount_2)
print(f"Итоговая сумма для {amount_2} со скидкой {discount_2*100}%: {final_amount_2}")
# Ожидаемый вывод: Итоговая сумма для 200 со скидкой 15.0%: 170.0
```

*Пояснение*: Функция вычтет 15% от `200` (что равно `30`) из исходной суммы, возвращая `170.0`.

### 3. Расчёт платежа с явно указанной нулевой скидкой

Вы также можете явно указать скидку `0`, чтобы подчеркнуть отсутствие скидки, хотя это и не обязательно, если вы не передаёте `discount` вовсе. Результат будет таким же, как в первом примере.

```python
# Пример 3: Платеж на 50 единиц с явно указанной нулевой скидкой
amount_3 = 50
discount_3 = 0  # Явно указываем отсутствие скидки
final_amount_3 = process_payment(amount_3, discount_3)
print(f"Итоговая сумма для {amount_3} с явно нулевой скидкой: {final_amount_3}")
# Ожидаемый вывод: Итоговая сумма для 50 с явно нулевой скидкой: 50.0
```

*Пояснение*: Этот пример демонстрирует, что передача `0` в `discount` даёт тот же результат, что и использование значения по умолчанию.

### 4. Работа с дробными суммами и скидками

Функция корректно обрабатывает суммы и скидки с плавающей точкой, что позволяет вам работать с точными финансовыми расчётами.

```python
# Пример 4: Платеж на 125.75 единиц со скидкой 5% (0.05)
amount_4 = 125.75
discount_4 = 0.05  # 5% скидки
final_amount_4 = process_payment(amount_4, discount_4)
print(f"Итоговая сумма для {amount_4} со скидкой {discount_4*100}%: {final_amount_4}")
# Ожидаемый вывод: Итоговая сумма для 125.75 со скидкой 5.0%: 119.4625
```

*Пояснение*: Обратите внимание, что результат может быть числом с множеством знаков после запятой. В реальных финансовых системах часто требуется округление до определённого количества десятичных знаков (например, до двух для валют). Для этого можно использовать встроенную функцию `round()` или модуль `decimal`.
